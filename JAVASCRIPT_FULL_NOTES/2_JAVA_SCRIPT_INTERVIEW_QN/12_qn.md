# JavaScript Interview Question

## 13. Explain the `this` keyword in JavaScript

### Definition

The `this` keyword in JavaScript refers to the **object that is currently executing the code**.  
Its value depends on **where and how the function is called**.

---

## Rules for `this` in JavaScript

### 1. Global Context

In the global scope, `this` refers to the **global object**.

```javascript
console.log(this);
```

- In browser → `window` object  
- In Node.js → `global` object

---

### 2. Inside a Function (Non-strict mode)

```javascript
function test() {
  console.log(this);
}

test();
```

- `this` refers to **global object** (window in browser)  
- In **strict mode**, `this` is **undefined**

---

### 3. Inside an Object Method

```javascript
let person = {
  name: "Rahul",
  greet: function() {
    console.log(this.name);
  }
};

person.greet();
```

Output:
```
Rahul
```

Explanation:  
- Here `this` refers to the **object `person`**.

---

### 4. Inside a Constructor Function

```javascript
function Person(name) {
  this.name = name;
}

let p1 = new Person("Amit");
console.log(p1.name);
```

- `this` refers to the **new object created** by the constructor.

---

### 5. Inside an Arrow Function

- Arrow functions **do not have their own `this`**.
- They inherit `this` from the **lexical scope** (the surrounding code).

```javascript
let obj = {
  num: 10,
  arrowFunc: () => console.log(this.num)
};

obj.arrowFunc(); // undefined
```

---

## Summary Table

| Context | Value of `this` |
|---------|----------------|
| Global | Global object (window in browser) |
| Function (non-strict) | Global object |
| Function (strict) | undefined |
| Object method | The object itself |
| Constructor | The new object created |
| Arrow function | Inherits from surrounding scope |

---

## Short Interview Answer

The `this` keyword in JavaScript refers to the **current executing object**, and its value depends on **how and where the function is called**.
