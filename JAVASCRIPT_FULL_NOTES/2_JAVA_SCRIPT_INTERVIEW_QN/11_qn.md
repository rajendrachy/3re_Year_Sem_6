# JavaScript Interview Question

## 11. What do you mean by Strict Mode in JavaScript and its Characteristics?

### Definition

**Strict Mode** in JavaScript is a **way to opt into a restricted variant of JavaScript**, which makes the code **more secure, easier to debug, and avoids common mistakes**.

It is enabled by adding:

```javascript
"use strict";
```

at the beginning of a script or function.

---

## How to Enable Strict Mode

1. **For the entire script**

```javascript
"use strict";

let x = 3.14;
```

2. **For a single function**

```javascript
function myFunction() {
  "use strict";
  let y = 3.14;
}
```

---

## Characteristics of Strict Mode

1. **Eliminates some silent errors**
   - Example: Assigning a value to an undeclared variable causes an error.

```javascript
"use strict";
x = 10; // ReferenceError
```

2. **Prevents accidental globals**
   - Variables must be declared using `var`, `let`, or `const`.

3. **Disallows duplicate parameter names**
   ```javascript
   function sum(a, a) { } // SyntaxError in strict mode
   ```

4. **Makes `this` undefined in functions**
   - In normal mode, `this` refers to global object (`window`)
   - In strict mode, `this` is `undefined` in functions

5. **Prevents deletion of variables**
   ```javascript
   "use strict";
   let a = 10;
   delete a; // SyntaxError
   ```

6. **Throws error on writing to read-only properties**

---

## Advantages of Strict Mode

- Helps **catch coding errors**
- **Improves performance** in some engines
- **Secures JavaScript code**
- Makes debugging **easier**

---

## Short Interview Answer

Strict Mode in JavaScript is a way to **enable stricter parsing and error handling** in your code, helping to catch mistakes, prevent unsafe actions, and write cleaner and more secure JavaScript programs.
