# JavaScript Interview Question

## 14. What do you mean by Self-Invoking Functions in JavaScript?

### Definition

A **Self-Invoking Function** is a function that **executes immediately after it is defined** without needing to be called explicitly.  

It is also called an **Immediately Invoked Function Expression (IIFE)**.

---

### Syntax

```javascript
(function() {
  // code to execute
})();
```

or using arrow function:

```javascript
(() => {
  // code to execute
})();
```

---

### Example

```javascript
(function() {
  console.log("This function runs automatically!");
})();
```

Output:

```
This function runs automatically!
```

Explanation:
- The function is **defined and executed immediately**
- No external function call is required

---

### Why Use Self-Invoking Functions

1. **Create private scope**  
   - Variables inside the function are not accessible from outside
2. **Avoid polluting global namespace**  
   - Useful in JavaScript to prevent conflicts
3. **Execute code immediately**  
   - Commonly used for initialization tasks

---

### Example with Private Variables

```javascript
(function() {
  let secret = "I am private";
  console.log(secret);
})();

console.log(secret); // Error: secret is not defined
```

---

### Short Interview Answer

A **Self-Invoking Function** in JavaScript is a function that **runs immediately after it is defined**, often used to create private variables and avoid global scope pollution.
