# 12. What is the Difference Between Synchronous and Asynchronous Code?

## Synchronous Code

- **Definition:** Executes line by line, one task at a time. Each operation waits for the previous one to finish before executing.
- **Characteristics:**
  - Simple and easy to read.
  - Can block the program if a task takes time.
- **Example:**
console.log("Start");
console.log("Middle");
console.log("End");
// Output:
// Start
// Middle
// End

## Asynchronous Code

- **Definition:** Executes tasks without waiting for previous tasks to complete. Allows other code to run while waiting for long operations (like API calls, timers, or file reads).
- **Characteristics:**
  - Non-blocking.
  - Can improve performance, especially for I/O operations.
- **Example:**
console.log("Start");
setTimeout(() => {
  console.log("Middle");
}, 2000);
console.log("End");
// Output:
// Start
// End
// Middle  (after 2 seconds)

## Key Differences

Feature | Synchronous | Asynchronous
--------|------------|-------------
Execution | Line by line | Tasks can run in parallel / non-blocking
Blocking | Yes | No
Complexity | Simple | Slightly complex (callbacks, promises, async/await)
Use Case | Small tasks, calculations | Network requests, file I/O, timers
