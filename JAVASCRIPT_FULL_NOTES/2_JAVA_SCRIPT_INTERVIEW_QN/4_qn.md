# JavaScript Interview Question

## 3. Why do we use the word “debugger” in JavaScript?

### Definition

The **debugger** keyword in JavaScript is used to **pause the execution of code** and allow developers to **inspect variables, functions, and program flow** to find and fix errors (bugs).

It works when the code is run in a browser or development tool that has debugging support.

---

## Syntax

```javascript
debugger;
```

---

## Example

```javascript
let x = 10;
let y = 20;

let sum = x + y;

debugger;

console.log(sum);
```

### What Happens

When JavaScript reaches the **debugger statement**:
- The program **stops execution**
- The browser **opens the debugging tool**
- Developers can **check variables and step through the code**

---

## Why We Use Debugger

- To **find errors (bugs)** in code
- To **pause code execution**
- To **inspect variable values**
- To **trace program execution step by step**

---

## Example Use Case

If your code is not working correctly, you can insert a **debugger statement** to see what values variables have at that point.

---

## Short Interview Answer

The **debugger** keyword in JavaScript is used to **pause code execution and open debugging tools**, allowing developers to inspect variables and find errors in the program.


# Meaning
- Inspect means to examine or check something carefully.
