# Creating a Custom AMI with Ubuntu OS and Nginx in AWS

This guide explains how to create a custom Amazon Machine Image (AMI) in AWS with Ubuntu OS and Nginx installed.

---

## Prerequisites

- An AWS account with permissions to launch EC2 instances and create AMIs.
- Basic knowledge of EC2 and SSH.
- AWS CLI installed (optional for automation).

---

## Step 1: Launch an Ubuntu EC2 Instance

1. Log in to the [AWS Management Console](https://aws.amazon.com/console/).
2. Navigate to **EC2** service.
3. Click **Launch Instance**.
4. Choose **Ubuntu Server 22.04 LTS** (or desired version) from the **Amazon Machine Images (AMI)** list.
5. Select an **Instance Type** (e.g., t2.micro for testing).
6. Configure instance details as needed.
7. Add storage (default is usually sufficient).
8. Add tags if required.
9. Configure security group:
    - Allow **SSH (port 22)**.
    - Allow **HTTP (port 80)** for Nginx.
10. Review and launch the instance.
11. Choose or create a **key pair** to access the instance.

---

## Step 2: Connect to the EC2 Instance

1. Open terminal (Linux/macOS) or use PuTTY (Windows).
2. Connect using SSH:

```bash
ssh -i /path/to/key.pem ubuntu@<EC2_PUBLIC_IP>
```

3. Verify connectivity.

---

## Step 3: Update the System

Once connected, update packages:

```bash
sudo apt update && sudo apt upgrade -y
```

---

## Step 4: Install Nginx

Install Nginx web server:

```bash
sudo apt install nginx -y
```

Start and enable Nginx service:

```bash
sudo systemctl start nginx
sudo systemctl enable nginx
```

Verify installation by opening your EC2 public IP in a browser: `http://<EC2_PUBLIC_IP>`

---

## Step 5: Clean Up the Instance (Optional)

Before creating an AMI, you can clean up temporary files:

```bash
sudo apt autoremove -y
sudo apt clean
```

---

## Step 6: Create a Custom AMI

1. Go to **EC2 Dashboard** → **Instances**.
2. Select your Ubuntu instance.
3. Click **Actions** → **Image and templates** → **Create image**.
4. Fill in details:
    - **Image name**: e.g., `Ubuntu-Nginx-AMI`
    - **Image description**: Optional.
    - **No reboot**: leave unchecked (recommended) to ensure filesystem consistency.
5. Click **Create Image**.

AWS will now create a custom AMI.

---

## Step 7: Launch Instances Using the Custom AMI

1. Navigate to **AMIs** in the EC2 Dashboard.
2. Select your custom AMI.
3. Click **Launch instance from AMI**.
4. Configure instance type, network, storage, and security group.
5. Launch the instance.
6. Connect via SSH to verify Nginx is installed and running.

---

## Step 8: Optional Automation with AWS CLI

You can create an AMI using AWS CLI:

```bash
aws ec2 create-image --instance-id <INSTANCE_ID> --name "Ubuntu-Nginx-AMI" --description "Custom Ubuntu with Nginx" --no-reboot
```

Verify AMI creation:

```bash
aws ec2 describe-images --owners self
```

---

## Summary

You have successfully:

- Launched an Ubuntu EC2 instance.
- Installed Nginx.
- Created a custom AMI.
- Launched new instances from the custom AMI.

This custom AMI can now be reused to quickly deploy multiple instances with Nginx pre-installed.
