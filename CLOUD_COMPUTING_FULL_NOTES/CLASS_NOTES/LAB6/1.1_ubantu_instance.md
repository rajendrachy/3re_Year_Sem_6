# AWS EC2 Guide: Creating Ubuntu Instances with Apache and Load Balancer

This guide explains how to create 3 Ubuntu EC2 instances running Apache and configure an Application Load Balancer with a Target Group to distribute traffic across the instances.

---

## Q1: Steps to Create 3 Ubuntu EC2 Instances with Apache and Configure ALB

### Step 1: Launch 3 Ubuntu EC2 Instances

1. Log in to the [AWS Management Console](https://aws.amazon.com/console/).
2. Navigate to **EC2 → Instances → Launch Instance**.
3. Select **Ubuntu Server 22.04 LTS**.
4. Choose instance type (e.g., t2.micro).
5. Configure instance details and select the same VPC and subnets for all three instances.
6. Configure security group to allow **SSH (22)** and **HTTP (80)**.
7. Launch 3 instances with the same or different key pair.

### Step 2: Connect and Install Apache on Each Instance

Connect via SSH to each instance and run:

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install apache2 -y
sudo systemctl start apache2
sudo systemctl enable apache2
```

Verify Apache is running by accessing each instance's public IP in a browser: `http://<EC2_PUBLIC_IP>`

### Step 3: Create an Application Load Balancer (ALB)

1. Navigate to **EC2 → Load Balancers → Create Load Balancer → Application Load Balancer**.
2. Enter a name, select **internet-facing**, and IPv4.
3. Choose the VPC and subnets where the EC2 instances are deployed.
4. Configure a security group to allow **HTTP (80)** and **HTTPS (443)**.

### Step 4: Create a Target Group

1. While creating the ALB, create a **Target Group**.
2. Choose target type: **Instances**.
3. Protocol: **HTTP**, Port: 80.
4. Select **health check settings** (default HTTP /index.html).
5. Register the 3 EC2 instances to this Target Group.

### Step 5: Review and Create ALB

* Review settings and create the ALB.
* Note the **DNS name** of the ALB.

### Step 6: Verify Load Balancing

* Open a browser and navigate to the ALB DNS name: `http://<ALB_DNS_NAME>`
* Refresh multiple times to see traffic distributed across the three EC2 instances.
* Check **Target Group → Targets** in the console to verify instances are healthy and receiving traffic.

### Step 7: Optional CLI Commands

```bash
# Create target group
aws elbv2 create-target-group --name MyTargetGroup --protocol HTTP --port 80 --vpc-id <VPC_ID> --target-type instance

# Register instances with target group
aws elbv2 register-targets --target-group-arn <TARGET_GROUP_ARN> --targets Id=<INSTANCE_ID1> Id=<INSTANCE_ID2> Id=<INSTANCE_ID3>

# Create Application Load Balancer
aws elbv2 create-load-balancer --name MyALB --subnets <SUBNET1> <SUBNET2> --security-groups <SG_ID> --scheme internet-facing --type application

# Create listener for ALB
aws elbv2 create-listener --load-balancer-arn <ALB_ARN> --protocol HTTP --port 80 --default-actions Type=forward,TargetGroupArn=<TARGET_GROUP_ARN>
```

## Summary

You have successfully:

* Launched 3 Ubuntu EC2 instances.
* Installed and configured Apache web server.
* Created an Application Load Balancer.
* Configured a Target Group to distribute traffic across the instances.
