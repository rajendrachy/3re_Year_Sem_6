# AWS EC2 Guide: Custom AMI, Elastic IP, and Auto Scaling

This guide explains how to create a custom Amazon Machine Image (AMI) in AWS with Ubuntu OS and Nginx installed, how to allocate and attach an Elastic IP (EIP), and how to configure Auto Scaling with a Load Balancer for EC2 instances.

---

## Questions

1. How can we create a Custom AMI with Ubuntu OS and Nginx installed in AWS?
2. How can we create and attach an Elastic IP to an EC2 instance in AWS?
3. Explain how to configure Auto Scaling for EC2 instances in AWS with a Load Balancer.

---

## Part 1: Creating a Custom AMI with Ubuntu OS and Nginx

### Prerequisites

* An AWS account with EC2 and AMI permissions.
* Basic knowledge of EC2 and SSH.
* AWS CLI installed (optional).

### Step 1: Launch an Ubuntu EC2 Instance

1. Log in to the [AWS Management Console](https://aws.amazon.com/console/).
2. Navigate to **EC2** service.
3. Click **Launch Instance**.
4. Choose **Ubuntu Server 22.04 LTS**.
5. Select an **Instance Type** (e.g., t2.micro).
6. Configure instance details, storage, and tags as needed.
7. Configure security group: allow **SSH (22)** and **HTTP (80)**.
8. Launch instance with a key pair.

### Step 2: Connect to the EC2 Instance

```bash
ssh -i /path/to/key.pem ubuntu@<EC2_PUBLIC_IP>
```

### Step 3: Update the System

```bash
sudo apt update && sudo apt upgrade -y
```

### Step 4: Install Nginx

```bash
sudo apt install nginx -y
sudo systemctl start nginx
sudo systemctl enable nginx
```

Test Nginx via browser: `http://<EC2_PUBLIC_IP>`

### Step 5: Clean Up the Instance (Optional)

```bash
sudo apt autoremove -y
sudo apt clean
```

### Step 6: Create a Custom AMI

1. In EC2 Dashboard, select your instance.
2. Click **Actions → Image and templates → Create image**.
3. Enter name and description.
4. Click **Create Image**.

### Step 7: Launch Instances Using Custom AMI

1. Navigate to **AMIs**.
2. Select the AMI → **Launch instance from AMI**.
3. Configure instance and security settings.
4. Launch and connect to verify Nginx.

### Step 8: Optional Automation with AWS CLI

```bash
aws ec2 create-image --instance-id <INSTANCE_ID> --name "Ubuntu-Nginx-AMI" --description "Custom Ubuntu with Nginx" --no-reboot
aws ec2 describe-images --owners self
```

## Part 2: Allocate and Attach an Elastic IP (EIP) to an EC2 Instance

### Step 1: Allocate an Elastic IP

1. Navigate to **EC2 → Network & Security → Elastic IPs**.
2. Click **Allocate Elastic IP address**.
3. Select **Amazon's pool of IPv4 addresses**.
4. Click **Allocate**.

### Step 2: Attach Elastic IP to an EC2 Instance

1. Select the allocated Elastic IP.
2. Click **Actions → Associate Elastic IP address**.
3. Choose **Instance** and select your EC2 instance.
4. Optionally, select a **private IP**.
5. Click **Associate**.

### Step 3: Verify the Elastic IP

```bash
ssh -i /path/to/key.pem ubuntu@<ELASTIC_IP>
```

### Step 4: Optional AWS CLI Commands

```bash
aws ec2 allocate-address --domain vpc
aws ec2 associate-address --instance-id <INSTANCE_ID> --allocation-id <ALLOCATION_ID>
```

## Part 3: Configure Auto Scaling with a Load Balancer

### Step 1: Create a Launch Template

1. Navigate to **EC2 → Launch Templates**.
2. Click **Create launch template**.
3. Enter **Template Name** and **AMI ID** (your custom AMI).
4. Select instance type and key pair.
5. Configure security group (allow HTTP/HTTPS and SSH).
6. Click **Create launch template**.

### Step 2: Create an Application Load Balancer (ALB)

1. Navigate to **EC2 → Load Balancers**.
2. Click **Create Load Balancer → Application Load Balancer**.
3. Enter name, scheme (internet-facing), and IP type (IPv4).
4. Select VPC and subnets.
5. Configure security groups (allow HTTP/HTTPS).
6. Configure **Target Group**:

   * Target type: **Instances**
   * Protocol: **HTTP**
   * Port: 80
7. Click **Create**.

### Step 3: Create an Auto Scaling Group

1. Navigate to **EC2 → Auto Scaling Groups**.
2. Click **Create Auto Scaling group**.
3. Enter **Auto Scaling Group Name** and select the **Launch Template**.
4. Choose VPC and subnets.
5. Attach the **Target Group** from ALB.
6. Configure group size:

   * Minimum instances: 1
   * Desired instances: 2
   * Maximum instances: 5
7. Configure scaling policies (optional, e.g., scale out on CPU > 70%).
8. Click **Create Auto Scaling group**.

### Step 4: Verify Auto Scaling with Load Balancer

* Navigate to **Load Balancers → Targets**.
* Ensure EC2 instances are registered and healthy.
* Access ALB DNS name in a browser: `http://<ALB_DNS_NAME>`.
* Auto Scaling will automatically launch/terminate instances based on load.

### Step 5: Optional AWS CLI Commands

```bash
# Create launch template
aws ec2 create-launch-template --launch-template-name MyTemplate --version-description v1 --launch-template-data file://launch-template.json

# Create Auto Scaling group
aws autoscaling create-auto-scaling-group --auto-scaling-group-name MyASG --launch-template LaunchTemplateName=MyTemplate,Version=1 --min-size 1 --max-size 5 --desired-capacity 2 --vpc-zone-identifier "subnet-12345,subnet-67890" --target-group-arns <TARGET_GROUP_ARN>
```
