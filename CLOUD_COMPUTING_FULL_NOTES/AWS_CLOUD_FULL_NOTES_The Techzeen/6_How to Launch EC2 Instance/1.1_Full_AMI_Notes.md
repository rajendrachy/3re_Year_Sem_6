# What is AMI?

AMI stands for **Amazon Machine Image**.

An Amazon Machine Image (AMI) is a pre-configured template used to
launch an Amazon EC2 instance.

## 🧠 Simple Definition

AMI is like a ready-made blueprint or template of a computer that
includes the operating system and required software.

When we launch an EC2 instance, we must choose an AMI first.

## 📦 What Does an AMI Contain?

An AMI includes:

-   Operating System (Linux, Windows, etc.)
-   Application server (optional)
-   Required software packages
-   Configuration settings
-   Root volume (EBS snapshot)

## 🚀 How AMI Works

1.  Choose an AMI\
2.  Launch EC2 instance using that AMI\
3.  AWS creates a virtual machine based on that template

👉 Every EC2 instance is created from an AMI.

## 🏷 Types of AMIs

-   Public AMI -- Provided by AWS or community.
-   Private AMI -- Created and used by your own AWS account.
-   Marketplace AMI -- Paid AMIs with licensed software.

## 🖥 Example

If you select:

-   Ubuntu AMI → Your EC2 instance will run Ubuntu OS.
-   Windows AMI → Your EC2 will run Windows Server.

## 🎯 Easy Example to Remember

-   EC2 = Computer\
-   AMI = Operating system + software template\
-   EBS = Hard disk\
-   Security Group = Firewall
