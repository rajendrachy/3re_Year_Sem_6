# 2️⃣ ISO 27017 – Cloud Security Guidelines (02:08)

## 🔹 What is ISO 27017?
ISO 27017 is an international standard that provides security controls specifically for cloud computing.

It is an extension of ISO 27001 and focuses on cloud-specific risks.

---

## 🔹 Purpose of ISO 27017

✔ Defines roles and responsibilities between cloud provider and customer  
✔ Provides guidelines for secure cloud implementation  
✔ Helps reduce cloud security risks  

---

## 🔹 Key Focus Areas

- Shared responsibility model  
- Virtual machine security  
- Cloud service agreements  
- Monitoring and logging  

👉 It improves trust between cloud providers and customers.

---

# 3️⃣ ISO 27018 – Protection of Personal Data in the Cloud (03:40)

## 🔹 What is ISO 27018?
ISO 27018 focuses on protecting Personally Identifiable Information (PII) in cloud environments.

It ensures privacy protection when personal data is stored in the cloud.

---

## 🔹 Key Objectives

✔ Protect customer data privacy  
✔ Prevent unauthorized data usage  
✔ Ensure transparency in data processing  
✔ Control data deletion and return  

---

## 🔹 Why Important?

- Helps organizations handle personal data securely  
- Builds customer trust  
- Supports compliance with global privacy laws  

---

# 4️⃣ NIST SP 800-53 & CSA Controls (04:41)

## 🔹 NIST SP 800-53

NIST Special Publication 800-53 provides a catalog of security and privacy controls for information systems.

It is widely used by government agencies and enterprises.

### Key Features:
✔ Risk-based approach  
✔ Security control families  
✔ Continuous monitoring  
✔ Access control guidelines  

---

## 🔹 CSA (Cloud Security Alliance) Controls

CSA provides the Cloud Controls Matrix (CCM).

It is specifically designed for cloud security.

### Key Benefits:
✔ Cloud-focused security framework  
✔ Aligns with other standards  
✔ Helps in cloud risk assessment  

---

## 🔹 Difference Between NIST & CSA

- NIST = General security control framework  
- CSA = Cloud-specific security framework  

Both are widely adopted in enterprises.

---

# 5️⃣ Global Data Protection Regulations (06:16)

Different countries have legal regulations to protect data.

---

## 🔹 GDPR (General Data Protection Regulation)

- Applicable in European Union  
- Protects personal data of EU citizens  
- Requires strict data privacy compliance  

Key Principles:
✔ Data minimization  
✔ Consent requirement  
✔ Right to be forgotten  
✔ Breach notification  

---

## 🔹 HIPAA (Health Insurance Portability and Accountability Act)

- Applicable in United States  
- Protects healthcare data  
- Used in hospitals and insurance companies  

Requires:
✔ Secure storage of medical records  
✔ Strict access control  
✔ Data confidentiality  

---

## 🔹 Other Regulations

- PCI-DSS (Payment Card Industry Data Security Standard)  
- SOC 2 (Service Organization Control)  
- Local data protection laws  

---
