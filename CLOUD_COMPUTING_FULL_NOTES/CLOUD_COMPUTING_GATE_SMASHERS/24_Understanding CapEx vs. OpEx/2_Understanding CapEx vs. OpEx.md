# Cloud Computing  
## Lec-23: Cost Models – CapEx vs OpEx & Consumption-Based Pricing

When organizations move to the cloud, they also change **how they spend money on IT infrastructure**.  
Understanding **CapEx (Capital Expenditure)** and **OpEx (Operational Expenditure)** helps businesses manage costs efficiently.

Cloud computing mainly follows an **OpEx + consumption-based pricing model**.

---

# 1. What is CapEx (Capital Expenditure)?

**CapEx** refers to **large upfront investments in physical infrastructure**.

Companies purchase hardware and resources that they own and maintain.

### Examples

- Buying servers
- Building data centers
- Purchasing networking hardware
- Storage systems

### Characteristics of CapEx

- Large **initial investment**
- Infrastructure **owned by the company**
- Requires **maintenance and upgrades**
- Long-term planning required

### Example

A company builds its own data center:

```
Buy Servers
Buy Cooling Systems
Install Networking Equipment
Hire IT staff
```

This requires **high initial capital investment**.

---

# 2. What is OpEx (Operational Expenditure)?

**OpEx** refers to **ongoing operational costs** for services that are used regularly.

Instead of buying infrastructure, companies **pay for services as they use them**.

### Examples

- Cloud servers (AWS EC2)
- Cloud storage
- Database services
- Software subscriptions

### Characteristics of OpEx

- **No large upfront investment**
- Pay on **monthly or usage basis**
- Infrastructure managed by provider
- Scalable and flexible

### Example

Using a cloud server:

```
AWS EC2 instance
Pay monthly or per hour
```

The company **does not buy hardware**.

---

# 3. CapEx vs OpEx Comparison

| Feature | CapEx | OpEx |
|------|------|------|
| Investment | Large upfront cost | Pay as you go |
| Ownership | Company owns infrastructure | Cloud provider owns infrastructure |
| Maintenance | Company responsible | Provider responsible |
| Scalability | Difficult to scale | Easy to scale |
| Risk | High financial risk | Lower risk |

---

# 4. Consumption-Based Pricing in Cloud

Cloud computing uses a **consumption-based pricing model**.

This means:

> You only pay for the resources you use.

It is similar to **electricity or water bills**.

### Example

If you run a cloud server:

```
Running 1 hour → Pay for 1 hour
Running 10 hours → Pay for 10 hours
```

This provides **cost flexibility**.

---

# 5. Examples of Consumption-Based Services

Common cloud services using this model:

### Compute

- Pay per hour for virtual machines

### Storage

- Pay per GB stored

### Data Transfer

- Pay per GB of network usage

### Database

- Pay for storage + requests

---

# 6. Control vs Cost in Cloud Models

### CapEx Model

Advantages:

- Full control over infrastructure
- No dependency on provider

Disadvantages:

- High upfront cost
- Maintenance responsibility
- Difficult scaling

---

### OpEx / Cloud Model

Advantages:

- No large upfront investment
- Highly scalable
- Pay only for usage

Disadvantages:

- Less physical control
- Continuous operational cost

---

# 7. Why Cloud Uses OpEx

Cloud computing uses OpEx because it allows:

- Flexible resource usage
- Reduced infrastructure cost
- Faster deployment
- Easy scalability

Companies can **start small and scale as needed**.

---

# Real-Life Example

Traditional IT (CapEx):

```
Company buys 100 servers
Even if only 30 are used
Cost remains the same
```

Cloud (OpEx):

```
Company starts with 5 servers
Increase to 50 during high traffic
Reduce again later
Pay only for usage
```

---

# Key Points to Remember

- **CapEx = Large upfront investment**
- **OpEx = Ongoing operational cost**
- Cloud computing mainly follows **OpEx**
- Consumption-based pricing means **pay only for what you use**

---

# Summary

- CapEx involves **buying and owning infrastructure**.
- OpEx involves **paying for services when needed**.
- Cloud computing uses **OpEx with consumption-based pricing**.
- This model improves **flexibility, scalability, and cost efficiency**.
