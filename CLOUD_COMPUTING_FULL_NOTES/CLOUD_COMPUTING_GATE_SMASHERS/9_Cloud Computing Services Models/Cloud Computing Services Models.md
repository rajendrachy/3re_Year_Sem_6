# 1️⃣ Introduction to Cloud Service Models (00:11)

Cloud computing provides different types of services based on user needs. These services are divided into three major models:

1. **SaaS – Software as a Service**  
2. **PaaS – Platform as a Service**  
3. **IaaS – Infrastructure as a Service**  

👉 These models define **who manages what** in the cloud environment.

The main difference between them is the level of control and responsibility given to the user.

---

# 2️⃣ SaaS – Software as a Service (01:30)

## 🔹 What is SaaS?
SaaS is a cloud service model where users access software applications over the internet without managing infrastructure or platforms.

In SaaS:
- The cloud provider manages everything.
- The user just uses the application.

---

## 🔹 What Provider Manages in SaaS?
✔ Infrastructure  
✔ Servers  
✔ Networking  
✔ Storage  
✔ Runtime  
✔ Application updates  
✔ Security patches  

User Responsibility:
✔ Only use the software  
✔ Manage data inside the application  

---

## 🔹 Examples of SaaS
- Gmail  
- Google Docs  
- Microsoft 365  
- Zoom  

---

## 🔹 Advantages of SaaS
✅ No installation required  
✅ Accessible from anywhere  
✅ Automatic updates  
✅ Low maintenance  

---

# 3️⃣ PaaS – Platform as a Service (02:29)

## 🔹 What is PaaS?
PaaS provides a platform that allows developers to build, test, and deploy applications without managing the underlying infrastructure.

In PaaS:
- The provider manages infrastructure.
- The developer manages the application and code.

---

## 🔹 What Provider Manages in PaaS?
✔ Servers  
✔ Networking  
✔ Storage  
✔ Operating System  
✔ Runtime environment  

User Responsibility:
✔ Application code  
✔ Application data  

---

## 🔹 Examples of PaaS
- Google App Engine  
- Microsoft Azure App Service  
- Heroku  

---

## 🔹 Advantages of PaaS
✅ Faster development  
✅ No server management  
✅ Built-in development tools  
✅ Scalable environment  

---

# 4️⃣ IaaS – Infrastructure as a Service (03:34)

## 🔹 What is IaaS?
IaaS provides virtualized computing resources over the internet.

In IaaS:
- The provider gives infrastructure.
- The user manages OS, runtime, and applications.

---

## 🔹 What Provider Manages in IaaS?
✔ Physical servers  
✔ Networking  
✔ Virtualization  
✔ Data center  

User Responsibility:
✔ Operating System  
✔ Middleware  
✔ Runtime  
✔ Applications  
✔ Data  

---

## 🔹 Examples of IaaS
- AWS EC2  
- Microsoft Azure Virtual Machines  
- Google Compute Engine  

---

## 🔹 Advantages of IaaS
✅ High flexibility  
✅ Full control  
✅ Custom configuration  
✅ Scalable infrastructure  

---

# 5️⃣ Pizza Analogy to Compare SaaS, PaaS, IaaS (06:31)

Pizza example helps understand responsibilities clearly.

## 🍕 Traditional On-Premise
You:
- Buy ingredients
- Make dough
- Bake pizza
- Serve
- Clean up

👉 You manage EVERYTHING.

---

## 🍕 IaaS Model
Cloud provider gives:
- Oven
- Kitchen

You:
- Bring ingredients
- Prepare pizza
- Bake and serve

👉 Shared responsibility.

---

## 🍕 PaaS Model
Cloud provider gives:
- Oven
- Kitchen
- Dough
- Sauce

You:
- Add toppings
- Bake and serve

👉 Less responsibility than IaaS.

---

## 🍕 SaaS Model
Cloud provider:
- Makes pizza
- Bakes it
- Serves it

You:
- Just eat 🍕

👉 Maximum convenience, minimum responsibility.

---

# 📊 Quick Comparison Table

| Feature | SaaS | PaaS | IaaS |
|----------|------|------|------|
| User Control | Very Low | Medium | High |
| Provider Responsibility | Maximum | High | Basic Infrastructure |
| Best For | End Users | Developers | IT Admins |
| Example | Gmail | App Engine | EC2 |

---

# 📌 Summary

✔ SaaS = Ready-to-use software  
✔ PaaS = Platform for developers  
✔ IaaS = Virtual infrastructure  

👉 As control increases, responsibility also increases.
