## 15. What is the `useContext` Hook and when should you use it?

### What is `useContext`?

`useContext` is a React Hook that allows components to **access data from a React Context directly** without passing props manually through every level of the component tree.

It helps solve the problem of **prop drilling**, where props need to be passed through many intermediate components.

### Syntax

```javascript
const value = useContext(ContextName);
```

* **ContextName** – the context created using `React.createContext()`.
* **value** – the current context value provided by the nearest `Provider`.

### When Should You Use `useContext`?

You should use `useContext` when:

* Many components need access to the **same data**.
* You want to avoid **passing props through multiple levels**.
* Managing **global-like data** such as:

  * Theme (dark/light mode)
  * User authentication
  * Language settings
  * Application settings

### Example

```javascript
import React, { createContext, useContext } from 'react';

const UserContext = createContext();

function Child() {
  const user = useContext(UserContext);
  return <h1>Hello, {user}</h1>;
}

function App() {
  return (
    <UserContext.Provider value="Alice">
      <Child />
    </UserContext.Provider>
  );
}

export default App;
```

### Output

```
Hello, Alice
```

### Key Points

* `useContext` allows components to **consume context values easily**.
* It removes the need to pass props through multiple components.
* Works only with contexts created using `React.createContext()`.
