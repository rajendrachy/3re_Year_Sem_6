## 24. What is the Difference Between Client State and Server State?

In modern web applications, data can be managed either on the **client side (browser)** or on the **server side**. These are known as **Client State** and **Server State**.

### Client State
Client state is the data that is **stored and managed in the frontend application** (inside React components).

**Examples:**
- Form input values
- UI states (modal open/close)
- Theme (dark/light mode)
- Local counters

Client state is usually managed using:
- `useState`
- `useReducer`
- Context API
- State libraries like Redux or Zustand

### Server State
Server state is the data that **comes from a backend server or API** and needs to be fetched and synchronized with the client.

**Examples:**
- User data from a database
- Product lists from an API
- Blog posts from a server

Server state is commonly handled using:
- `useEffect` with `fetch` or `axios`
- Libraries like **React Query** or **SWR**

### Key Differences

| Feature | Client State | Server State |
|-------|--------------|--------------|
| Location | Stored in the browser | Stored on a remote server |
| Source | Created and managed by the frontend | Fetched from APIs or databases |
| Examples | UI state, form inputs | User data, product lists |
| Management | useState, useReducer, Context | API calls, React Query, SWR |

### Summary
- **Client State** manages local UI-related data inside the application.
- **Server State** represents data fetched from and synchronized with a backend server.
